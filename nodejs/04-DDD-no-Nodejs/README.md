# DDD (Domain-driven Design)

Design dirigido à domínio

### Domínio

- Domain Experts (Quem contem o domínio do assunto)
  - Conversa
- Linguagem ubíqua

- Usuário
  - Cliente
  - Fornecedor
  - Atendente
  - Barman

- Agregados
- Value Objects
- Eventos de domínio
- Subdomínios (Bounded Contexts)
- Entidades (tudo que factivel, tudo que vai manter na aplicação...)
- Casos de uso
